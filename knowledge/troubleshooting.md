## BUG-001: Кнопка "Расскажи о дочке" молчит (2026-03-18)

**Симптом:** Юзер нажимает "Расскажи о дочке" — бот не отвечает. Статус "печатает" висит, потом пропадает.

**Причина:** `ask_claude()` рефакторнули — стал возвращать `ClaudeResponse` (dataclass) вместо строки. Хендлер `my_child.py` не обновили — передавал объект в `split_long_message()` и `db.add_message()`.

**Фикс:** `response` -> `result`, текст через `result.text`. Добавлен `db.add_token_usage()` и строка стоимости.

**Урок:** При изменении сигнатуры/возврата функции — grep все вызовы, обновить ВСЕ хендлеры. Чеклист:
- `grep -rn "ask_claude" handlers/` — все вызовы используют `.text`?
- Все хендлеры сохраняют usage в `db.add_token_usage()`?

## BUG-002: Контекст протекает между сессиями (2026-03-18)

**Симптом:** Юзер обсуждал бессонницу ночью. Утром спрашивает про прививки — бот тянет контекст бессонницы в ответ.

**Причина:** `get_last_messages()` брал последние 20 сообщений без учёта времени. Если юзер пишет редко, 2-дневный разговор попадал в контекст нового вопроса.

**Фикс:** Добавлен `session_gap_hours` (по умолчанию 4). Если между сообщениями прошло >4 часов — старые не включаются. Настраивается через `SESSION_GAP_HOURS` в .env.

**Урок:** История чата != контекст сессии. Всегда учитывай временные разрывы при формировании контекста для LLM.

## Чеклист перед коммитом

1. Изменил возврат функции? -> grep все вызовы
2. Добавил поле в Config? -> добавь в load_config() + .env.example
3. Изменил схему БД? -> миграция или init_db()
4. Новый хендлер? -> добавь в main.py в правильном порядке (специфичные до chat)
5. Работа с Claude API? -> сохраняй usage, показывай стоимость
